**ID:** STORY-019
**Title:** Natural language task input parser
**Description:** The ⌘N 'Add task' field should accept natural language like 'Biology essay due Friday importance 8' or 'Maths test tomorrow' and auto-parse into TaskItem fields (title, dueDate, importance, subject) using a local regex/keyword parser — no AI needed. Unparsed tokens become the title.

**Acceptance criteria:**
- Input 'Economics test due Friday importance 10' creates a task with dueDate=next Friday, importance=10, subject guessed as 'Economics'
- Input 'Maths homework tomorrow' creates task with dueDate=tomorrow, subject='Maths'
- Input 'Read chapter 3' creates task with title 'Read chapter 3', no date, importance=5 default
- Parsed fields shown as inline chips below the input before confirming (e.g. '📅 Fri', '⭐ 10', '📚 Economics')
- Parser handles: due/by/on + (today|tomorrow|monday..sunday|next week), importance/priority + digit, subject keywords matching existing subjects
- swift build passes

**Implementation notes:**
Create NLTaskParser.swift with a `parse(_ input: String) -> ParsedTask` struct. Use NSDataDetector for date extraction OR a manual regex: `(due|by|on)\s+(today|tomorrow|monday|tuesday|wednesday|thursday|friday|saturday|sunday|next week)`. For importance: `(importance|priority|imp)\s*(\d+)`. For subject: check input.lowercased() against known subjects ['economics','english','maths','biology','society','history','physics','chemistry']. Return a ParsedTask with optional fields. In ContentView, show chip previews in the add-task sheet using HStack of small rounded Text badges.
