**ID:** STORY-011
**Title:** Full-text context search across all imported content
**Description:** Add a search bar in the right panel (or as a command palette) that searches across all imported content: Seqta tasks, Obsidian notes, transcript excerpts, codex-mem hits, and chat history. Returns ranked results with source labels.

**Acceptance criteria:**
- A search field (⌘K keyboard shortcut to focus) is accessible from the main layout — either in the toolbar or at the top of the right panel
- Typing triggers real-time search across: PlannerTask titles/descriptions, ContextDocument content, transcript excerpts, and chat messages
- Results are grouped by source type (Tasks, Notes, Transcripts, Chat)
- Each result shows: title, source type icon, subject badge, first matching sentence highlighted
- Clicking a result: if task → select task and show in right panel; if note → open note preview in right panel; if chat message → scroll chat to that message
- Search is client-side (no network) and handles up to 10,000 documents without lag
- App builds without errors

**Implementation notes:**
Create a ContextSearchService.swift with: `func search(_ query: String, in store: PlannerStore) -> [SearchResult]`. SearchResult: { id, type: .task/.note/.transcript/.chat, title, subject, snippet, relevanceScore }. Scoring: exact title match = 100, substring match in title = 80, content match = 50, weighted by recency. For performance: build an in-memory index at startup — Dict<String, [SearchResult]> keyed by first 3 chars of each word. For the UI: create SearchPanelView.swift with a focused TextField, results List grouped by type. Wire ⌘K using .keyboardShortcut('k', modifiers: .command) on a hidden Button.
