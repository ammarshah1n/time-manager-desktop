## Current story

**ID:** STORY-008
**Title:** Focus timer — inline Pomodoro with task binding
**Description:** Add a focus timer that binds to a specific task. When started, a small timer widget appears in the center column above the command composer. Timer counts down (25min default, configurable). On completion: plays a sound, marks a pomodoro on the task, prompts for a 5-min break. After 4 pomodoros: long break prompt.

**Acceptance criteria:**
- Each task row in the ranked list has a small ▶ play button — clicking it starts a focus session bound to that task
- A timer widget appears at the top of the center column showing: task title (truncated), time remaining (MM:SS), a thin progress ring, pause/stop buttons
- Timer duration is configurable in Settings (default 25min, options: 15, 25, 45, 60)
- On timer completion: NSSound plays a soft chime (use NSSound.beep() or a bundled .mp3), a notification is posted via UNUserNotificationCenter
- Completed pomodoro is recorded on the task (task.pomodoroCount += 1)
- After 4 pomodoros on any task: show 'Take a long break (15 min)?' prompt as a banner
- App builds without errors

**Implementation notes:**
Create FocusTimerModel.swift: @Observable class with currentTask: PlannerTask?, secondsRemaining: Int, isRunning: Bool, pomodoroCount: Int. Use Timer.scheduledTimer or Swift Concurrency (Task { while isRunning { try await Task.sleep(nanoseconds: 1_000_000_000); secondsRemaining -= 1 } }). FocusTimerWidget.swift: a compact HStack showing task name, circular progress (drawn with a SwiftUI Circle stroke with trim()), and controls. Inject FocusTimerModel as @State in ContentView and pass it down. For notifications: request UNUserNotificationCenter.current().requestAuthorization(options: [.alert, .sound]) on first launch and trigger a UNMutableNotificationContent when timer hits 0. Store pomodoroCount in PlannerTask model (add field to Models.swift and PlannerSnapshot).
