### Fix PRs (#10–#14) — 5 bugs caught at execution time

Lexical reviewers (sqlparse, tsc, structural review) missed all 5. Each only surfaced when we ran the actual pipeline.

| PR | Bug |
|----|-----|
| #10 | Schema-version prefix collision — A1 + A3 both picked `20260425000000_*`. Supabase tracks by version key alone, so the second one fails on `INSERT INTO supabase_migrations.schema_migrations ... ON CONFLICT` duplicate-key. Fixed by bumping `executive_profile` prefix |
| #11 | `\|\|` string concat inside `COMMENT ON COLUMN ... IS` — Postgres rejects (COMMENT requires a single string literal). sqlparse accepts it |
| #12 | Same pattern in 3 more places (B1+B3) |
| #13 | A2's `services/graphiti/requirements.txt` pinned `neo4j==5.20.0`, but `graphiti-core==0.3.14` requires `neo4j>=5.23.0`. Pip refused to resolve |
| #14 | A2's `services/graphiti/app/main.py:31` imported `graphiti_core.cross_encoder.openai_reranker_client.OpenAIRerankerClient` — that path doesn't exist in graphiti-core 0.3.14. Container crash-loops on startup. Fixed by bumping to 0.11.6 |

**Systemic lesson**: pre-merge gate must include real-Postgres `--dry-run` against a scratch DB, `pip install --dry-run`, `docker build` for every Dockerfile in the diff, and a cross-PR schema-version-collision audit when multiple PRs touch the migrations directory.

## Lessons captured

In `.learnings/LEARNINGS.md` already: LRN-20260418-001 / -002 (assets / icon glob) and LRN-20260424-001 through -007 (ElevenLabs Custom LLM quirks, Anthropic max/budget rule, prompt-caching 1024-token minimum, Opus-wraps-JSON-in-prose, LiveKit embed rule, Haiku drift on multi-step prompts, permission-check hook bare-command gap).

From the Wave 1+2 retro (in `docs/sessions/2026-04-25-wave-1-2-shipped.md`):
1. Pre-assign schema-version timestamp slots before fanning out parallel agents — they can't see each other
2. Add a real-Postgres + real-Docker pre-merge gate. Lexical SQL parsing and tsc are not enough
3. Reviewer prompts should default to "execute, don't just read" — when SQL or a Dockerfile is in the diff, run `psql --check` / `pip install --dry-run` / `docker build` and report exit codes
4. Remote routines need self-contained prompts and explicit "you must produce a file" assertions, otherwise they fire and silently produce nothing
5. Trigger.dev `dev` mode + a hard-fail `defineConfig` can fight each other — c12 config loader runs before SDK loads `.env.local`. Source the env explicitly or move env-reads into a lazy function
