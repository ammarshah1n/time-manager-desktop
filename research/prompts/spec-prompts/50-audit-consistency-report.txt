**Your task: Read every document and produce a comprehensive consistency audit.**

Specifically, identify:

1. **Schema conflicts.** Do different documents define the same entity differently? Does the memory architecture spec define episodic memory one way while the reflection engine spec assumes a different structure? Does the CoreData migration strategy reference entities that don't appear in the memory architecture? Are there field names, types, or relationships that are inconsistent across documents?

2. **Interface mismatches.** Do the layer boundaries match? Does Layer 1's output format match what Layer 2 expects to receive? Does Layer 2's retrieval interface match what Layer 3 calls? Does Layer 3's reflection output match what Layer 4 needs to present? For every inter-layer interface: verify that both sides agree on the data contract.

3. **Undefined terms.** Terms used in one document that are never defined. Concepts referenced but never specified. Acronyms without expansion. Entity types mentioned in one spec but absent from the schema.

4. **Circular dependencies.** Components that depend on each other in ways that create cycles. Build phases that reference incomplete predecessors. Features that require features that require the first feature.

5. **Missing acceptance criteria.** Specs that describe what to build but not how to verify it works. ML models without quality thresholds. UI components without interaction specs. Agents without health check definitions. Integration points without error handling specs.

6. **Contradictions.** Direct conflicts between documents. One spec says data stays local, another implies it's synced. One spec says Haiku classifies emails, another says Sonnet. One spec says the reflection runs nightly, another implies on-demand. Find every contradiction.

7. **Gaps.** Things that aren't covered by any document but are obviously needed. Error handling that's unspecified. Edge cases not addressed. Platform requirements not documented. Performance targets not defined.

8. **Specification quality.** For each document, rate:
   - Completeness (does it cover everything it claims to?)
   - Implementability (could a developer build from this spec alone?)
   - Testability (are acceptance criteria concrete and measurable?)
   - Consistency with the overall architecture
