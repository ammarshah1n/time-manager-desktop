I'm building a macOS application called Timed — a cognitive intelligence layer for C-suite executives. It builds a compounding model of how a specific person thinks, decides, avoids, and operates. Observation and intelligence only — it never acts on the world.

**Hard constraints:**
- macOS native: Swift 5.9+, SwiftUI, CoreData, on-device voice
- Four layers: Signal Ingestion → Memory Store → Reflection Engine → Intelligence Delivery. No cross-layer coupling.
- Intelligence compounds — Month 6 >> Month 1
- Opus 4.6 for reflection, Sonnet for analysis, Haiku for classification
- Supabase + Edge Functions backend
- Microsoft Graph (MSAL OAuth) for calendar and email
- Apple Speech on-device for voice
- DMG + Sparkle for distribution
- Dev environment: Claude Code (Claude Max 20x), Comet MCP for browser automation
- Single developer (me) using AI-assisted development exclusively

**The full feature set to be built:**

**Layer 1 — Signal Ingestion:**
- Calendar observation agent (Microsoft Graph delta queries)
- Email observation agent (Microsoft Graph, metadata + classification)
- Voice capture agent (Apple Speech, on-device transcription)
- App focus agent (NSWorkspace + Accessibility API)
- Task completion agent (internal task system monitoring)

**Layer 2 — Memory Store:**
- Episodic memory (raw events with embeddings, Jina 1024-dim)
- Semantic memory (learned facts about the person, with confidence and decay)
- Procedural memory (operating rules generated by reflection)
- MemGPT-style context management (active/archival/retrieval tiers)
- Vector similarity search for memory retrieval

**Layer 3 — Reflection Engine:**
- Nightly Opus reflection cycle (raw observations → first-order patterns → second-order synthesis → rules → model update)
- Pattern detection (avoidance, energy, relationships, decision quality)
- Rule generation and confidence tracking
- Contradiction detection and resolution
- Model versioning (snapshot the cognitive model state over time)

**Layer 4 — Intelligence Delivery:**
- Morning session (voice-first briefing with named patterns, task plan, proactive insights)
- Menu bar presence (glanceable status, quick capture, focus state)
- Proactive alerts (pattern-triggered notifications)
- Focus timer (with completion data feeding back to Layer 2)
- Query interface ("why did you rank this task first?", "what patterns do you see in my meetings?")

**ML Models:**
- Thompson sampling for task scoring
- EMA for time estimation
- Chronotype detection
- Avoidance pattern detection
- Cognitive load estimation
- Relationship health scoring

**Infrastructure:**
- CoreData schema with migration strategy
- Supabase backend (auth, subscription, anonymised sync)
- Microsoft Graph OAuth integration
- Claude API integration (three-tier: Haiku/Sonnet/Opus)
- Jina embedding API integration
- DMG packaging and Sparkle auto-update
- Background agent architecture (XPC or equivalent)
- Privacy abstraction pipeline

**What I need:**

1. **Order these into phases where each phase produces a working increment.** Not a waterfall plan — each phase must produce something I can use and test on my own Mac. The first phase should produce a working app within 1-2 weeks. Each subsequent phase adds a capability layer. Dependencies must be respected (you can't build reflection without memory, you can't build memory without signals).

2. **For each phase, specify:**
   - What gets built (concrete deliverables)
   - What it depends on (which previous phases must be complete)
   - Completion criteria (how I know it's done — specific, testable)
   - Session estimate (in Claude Code sessions, assuming ~4-6 hours of focused work per session with AI assistance)
   - What "done" looks like to the user (what they experience)
   - Risk factors (what could go wrong, what's technically uncertain)

3. **The critical path.** Which phases block everything downstream? Where are the parallelisation opportunities? What can be built independently and integrated later?

4. **The "intelligence bootstrap" milestone.** At which phase does the system transition from "a nice macOS app" to "a system that is genuinely learning about the user"? This is the most important milestone. Everything before it is infrastructure. Everything after it is compounding returns.

5. **Testing strategy per phase.** What to test, how to test it, what constitutes a passing integration test for each phase. How to test the reflection engine's output quality (not just that it runs, but that it produces genuine insight). How to test that intelligence is actually compounding.

6. **The "ship to self" checkpoints.** At which phases can I start using this as my own daily driver? What's the minimum viable daily-use configuration? How to dogfood effectively when the product is designed for someone with a different work pattern (I'm a 16-year-old developer, not a CEO)?

7. **Dependency graph.** A clear map of what depends on what. Which components are leaf nodes (no dependencies on other Timed components)? Which are the integration bottlenecks? Where should I build interfaces/protocols first so that phases can be worked on out of order?

Produce a concrete, session-estimated build plan that a solo developer using Claude Code can follow. Not aspirational — realistic. Account for the fact that AI-assisted development is 3-5x faster than traditional development for well-specified tasks, but debugging and integration are not faster.

If current research suggests a superior approach to phased delivery of complex systems, recommend it with evidence.
