I'm building a macOS executive intelligence system called Timed that observes a C-suite executive's digital behaviour and builds a compounding cognitive model of them over months. It never acts on the world — observation and intelligence only.

I need a complete implementation spec for inferring the executive's actual chronotype — their peak cognitive windows per task type — from observed behavioural data. Not self-report. Not surveys. Purely from what the system sees them do.

Architecture context: the system has four decoupled layers — Signal Ingestion, Memory Store, Reflection Engine, Intelligence Delivery. This model sits at the boundary of Signal Ingestion and the Reflection Engine. Tech stack: Swift 5.9+, SwiftUI, CoreData on-device, Supabase backend, Jina embeddings (1024-dim), Claude Haiku/Sonnet/Opus tiered processing. The nightly reflection engine runs Opus at maximum effort with no cost cap. Intelligence must compound — month 6 should be incomparably better than month 1.

For each signal: what resolution is needed (minute, 15-min, hourly), what's the noise floor, what research validates its use as a cognitive performance proxy?

2. **Task type taxonomy for chronotype mapping** — Not all tasks have the same cognitive profile. Strategic/creative work, administrative triage, communication (email), analytical work, people/relationship work, decision-making. Research on which task types show the strongest time-of-day effects. How to classify the executive's activities into these categories from observed signals without explicit labelling.
